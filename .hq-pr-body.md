# fix(installer): eliminate silent crash + console hand-off on first run

## What's broken today

A fresh-machine test (bscott@twgsecurity.com, 2026-07-20) surfaced a completely
broken first run: the downloaded installer showed a spinner and then nothing, no
Desktop shortcuts appeared, the Start Menu entry had no icon, and clicking it
dropped straight into a raw PowerShell console with manual CLI instructions.

Root cause: two installers (the Electron `.exe` and the CLI `install.ps1`/
`setup.ps1` flow) never actually coordinated. The polished setup wizard's "Install
it for me" button on the brain step just opened a brand-new visible console
running the CLI installer — that console is exactly what a first-run user landed
in. Separately, the Electron backend spawn path had no error handler, so a spawn
failure (e.g. blocked by antivirus on an unfamiliar machine) crashed the whole app
silently instead of showing the app's own "backend failed to start" screen.

## What changed

- `desktop/src/main/backend.ts` — added the missing spawn-error handler; a backend
  spawn failure now surfaces the existing error screen instead of taking the whole
  process down with it.
- `backend/main.py` — new `/setup/install_brain` and `/setup/wizard/*` endpoints
  run the brain install (doctor → merge → memory_migrate) and the onboarding
  interview as background subprocesses, replacing the old `_spawn_visible_console`
  hand-off to a raw `install.ps1 | iex` console.
- `installer/wizard.py` — added a `--json` mode so the same interview (and its
  existing immediate per-answer file writes) can be driven by a UI instead of only
  a terminal `input()` loop. No reimplementation of the vault file formats.
- New onboarding UI (`frontend/src/components/onboarding/`) — a question-by-
  question setup screen inside the app, ending on a "LeRoy is ready to use" screen
  that launches the app. Replaces the old "a console window will open" hand-off.
- `installer/shortcuts.ps1` — now creates **both** "Leroy CLI" and "LeRoy UI"
  Desktop shortcuts (auto-detecting an existing LeRoy UI install), using the same
  Known-Folder-Move-safe resolution that previously only covered the CLI
  shortcut — this is why some machines showed zero or blank Desktop icons even
  when the installer reported success (OneDrive redirecting the Desktop folder).
- `desktop/package.json` / `build/installer.nsh` — electron-builder's own
  (unverified) NSIS shortcut creation is disabled in favor of the same
  `shortcuts.ps1`, invoked post-install with the just-installed app's real path.
- `bin/leroy.ps1` — `leroy start` now actually launches the installed LeRoy UI app
  instead of printing a stub message.
- Stale docs/comments corrected (`setup.ps1`, `shortcuts.ps1`, `leroy.ps1`,
  `README.md`) to match the above.

## Testing

  answered, all 9 expected vault files written, exit 0.
- `shortcuts.ps1` run against a fake Desktop dir: both shortcuts created, no
  duplicates on re-run, auto-detected an existing local LeRoy UI install.
- `leroy start`'s install-detection logic verified against a real local install.
- Full doctor → merge → memory_migrate → wizard chain run end-to-end against a
  scratch `~/.claude`: all steps exit 0, `brain_installed`/`brain_interview_done`
  (the same markers `/setup/status` already checks) both flip true.
- `desktop/` TypeScript type-checks clean (main + frontend packages).
- `frontend` production build (`vite build`, desktop mode) succeeds, 1046 modules,
  no errors.
- Not yet tested: a live NSIS install on a real fresh/OneDrive-redirected Windows
  account. That's the next step — release notes will be updated after Brian
  re-runs the original repro on the new machine.

## Safety attestation

- No client data, credentials, or private business handlers touched — this branch
  only edits the public CLI-installer scripts already in this repo.
- `installer/*.py` stay stdlib-only (no new dependency added).
